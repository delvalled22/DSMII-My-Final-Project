# DSMII-My-Final-Project


# Chicago Metro Accident Severity Classification

This project predicts whether a reported traffic accident in the Chicago metropolitan area of Illinois is likely to be **high severity**. It compares a **Random Forest classifier** and a **feedforward neural network** on the same binary classification problem so the final model choice is based on a fair, direct comparison.

The business goal is to support public-safety decision making. A useful severity classifier can help transportation departments, emergency-management teams, and traffic operations centers identify which incidents are more likely to require stronger operational attention.

## Dataset

**Source**: [US Accidents dataset on Kaggle](https://www.kaggle.com/datasets/sobhanmoosavi/us-accidents)

**Project scope**:
- Full local source file used for planning: `7,728,394` rows and `46` columns
- Illinois subset: `168,958` rows
- Chicago metropolitan subset: `145,845` rows
- Date range of Chicago subset: `2016-06-17` through `2023-03-31`
- Target variable: `High_Severity`
- Definition: `1` if `Severity` is `3` or `4`, `0` if `Severity` is `1` or `2`

**Chicago metro filter**:
- `Cook`
- `DuPage`
- `Kane`
- `Kendall`
- `Lake`
- `McHenry`
- `Will`
- `DeKalb`
- `Grundy`

**Relevant features include**:
- Time fields derived from `Start_Time`
- Location context such as `County`, `Start_Lat`, and `Start_Lng`
- Weather variables such as `Temperature(F)`, `Visibility(mi)`, `Wind_Speed(mph)`, and `Precipitation(in)`
- Road-context indicators such as `Junction`, `Traffic_Signal`, `Crossing`, and `Stop`
- Engineered features such as `rush_hour_flag`, `winter_flag`, `low_visibility_flag`, and `precipitation_x_visibility`

## Final Results

### Optimized Model Comparison

| Model | Accuracy | Precision | Recall | Training Time |
|---|---:|---:|---:|---:|
| Optimized Random Forest | 0.8213 | 0.7197 | 0.8278 | 28.2s |
| Optimized Neural Network | 0.7488 | 0.6131 | 0.8261 | 18.6s |

### Key Findings

- The **Random Forest** produced the strongest overall balance of accuracy, precision, and recall on the shared engineered feature set.
- The **Neural Network** remained competitive on recall, but lagged behind the forest on precision and overall stability.
- Feature engineering was still worthwhile: the engineered risk features improved the neural-network side more clearly than the random-forest side and made the feature space more interpretable.
- Nighttime conditions, weather context, traffic controls, temporal patterns, and spatial location all contributed meaningful predictive signal.

### Final Recommendation

The recommended deployment candidate is the **optimized Random Forest classifier** with a probability threshold of `0.40`. That threshold gives a better precision-recall balance for this public-safety use case than the default `0.50`, improving the model's ability to catch genuinely high-severity incidents without overwhelming operations with too many false alarms.

## Ethical and Practical Notes

This model is intended as a decision-support tool, not as a fully automated decision-maker. It should be used to assist human operators in triage and prioritization rather than replace judgment. Because the data reflects historical accident reporting and regional infrastructure differences, stakeholders should monitor subgroup performance, retrain periodically, and review threshold choices before any operational deployment.
